# NOVA-VAD Growth Plan

Objective: grow GitHub stars/forks and X followers by making NOVA-VAD easy to understand, easy to try, and worth discussing among builders working on audio ML, speech, ASR, diarization, edge AI, and voice agents.

## Positioning

Core line:

> NOVA-VAD is a lightweight, explainable voice activity detector for noisy real-world audio.

Proof line:

> On the current held-out noisy-audio benchmark, NOVA-VAD reports 93.0% accuracy / 92.63 F1 against WebRTC, Silero, and Pyannote baselines.

Use careful wording:

- Say "on our noisy-audio benchmark" instead of "best VAD."
- Say "no GPU required" and "lightweight" rather than implying zero heavy dependencies in every dev setup.
- Say "explainable feature-based decisions" instead of claiming perfect interpretability.

## Audience

- Voice-agent builders
- ASR and transcription developers
- Diarization/audio pipeline builders
- Edge AI developers
- Speech/audio ML researchers
- Open-source ML contributors

## Agent System

Leader Agent:

- Owns final judgment, approvals, and live account actions.
- Reviews every public post before publishing.

Product Agent:

- Maintains the strongest product explanation, benchmark caveats, and roadmap.

GitHub Conversion Agent:

- Improves README, issues, roadmap, contribution path, labels, releases, and topics.

Hugging Face Agent:

- Maintains model card, example usage, limitations, and links back to GitHub/X.

DevRel Content Agent:

- Writes X posts, threads, technical explainers, and demo captions.

Community Scout Agent:

- Finds relevant builders, repos, discussions, papers, and accounts.

Engagement Agent:

- Drafts helpful replies/comments. No generic praise. No link-dropping.

Analytics Agent:

- Tracks stars, forks, followers, impressions, replies, profile visits, and which angles work.

## 14-Day Launch Rhythm

Day 1:

- Pin core launch post on X.
- Add GitHub topics.
- Add README, roadmap, contribution, and issue templates.
- Link GitHub, Hugging Face, and X together.

Days 2-4:

- Post one technical proof point per day.
- Reply to 10 relevant audio/voice/ASR posts per day with useful context.
- Ask builders for hard noisy samples.

Days 5-7:

- Share one demo/explanation output.
- Share one failure-case request.
- Open 3-5 good first issues.

Week 2:

- Publish benchmark methodology post.
- Publish roadmap post.
- Publish "what I learned building explainable VAD" post.
- Thank contributors/star milestones with useful updates, not empty celebration.

## Daily Operating Checklist

- Check GitHub stars/forks/issues.
- Check X replies/mentions.
- Draft one useful public post.
- Draft 5-10 thoughtful replies.
- Convert repeated questions into README/model-card improvements.
- Record what worked in `docs/GROWTH_METRICS.md`.

## Engagement Rules

- Reply only where Nova-VAD is contextually relevant.
- Add a useful technical point before linking.
- Keep a 3:1 value-to-CTA ratio.
- Ask for tests and failure cases more often than stars.
- Do not mass DM.
- Do not tag large accounts unless replying to their actual post.
- Do not inflate claims beyond the benchmark.
